## 说明

1. 当前种子数据为每个组织都补齐了可登录测试账号。
2. 角色体系以审批工作流为准，当前使用 4 个 canonical 业务角色和 1 个系统角色：
   - `ROLE_REPORTER` (id=1) 填报人
   - `ROLE_APPROVER` (id=2) 部门负责人
   - `ROLE_STRATEGY_DEPT_HEAD` (id=3) 战略发展部负责人
   - `ROLE_VICE_PRESIDENT` (id=4) 分管校领导
   - `ROLE_SYSTEM_ADMIN` (id=5) 系统管理员
3. `admin` 是系统管理员专用账号，用于系统维护和导入后的系统自动审批，不作为人工业务审批候选人。
4. 职能部门默认提供 4 个账号：`_report`(填报人) / `_audit1`(部门负责人) / `_leader` 或 `_audit2`(分管校领导审批 seat，分配 `ROLE_VICE_PRESIDENT`) / `_reserve`(预留账号，无角色)
5. 二级学院默认提供 3 个账号：`_report`(填报人) / `_audit1`(部门负责人) / `_leader` 或 `_audit2`(分配 `ROLE_VICE_PRESIDENT`，用于学院院长审批 seat)
6. 战略发展部保留独立填报账号、真实分管校领导账号，以及 2 个分配 `ROLE_STRATEGY_DEPT_HEAD` 的审批账号。
6. **注意**：所有 `_reserve` 账号当前在 `sys_user_role` 中**无角色分配**，仅作为预留测试账号存在。
7. 部分早期组织（党委办、保卫处、马院）使用 `_audit2` 命名代替 `_leader`，角色分配相同；教务处额外保留一个 `jiaowu_reserve` 预留账号。
